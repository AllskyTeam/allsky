If you have a QUESTION or want to suggest a NEW FEATURE, add a new Discussion item via the link at the top of any GitHub page; do not submit an Issue.

If you found a BUG or something isn't working right, first look in the Discussions area and at existing Issues. If no one has already reported the problem, create a new Issue.

How to report an Issue

The automated method

This id the preferred method and should be used if you have access to the WebUI

Select the 'Getting Support' menu option from the WebUI. This will display the support log generation page

Towards the bottom of the page is a button that will generate a support file that you can attach to your discussion

Clicking the button will display a message describing what data will be included. Click ok and the newly generated support log will be displayed, you can generate multiple logs and the latest one is displayed in green if there is more than one log

If you know the discussion number from GitHub then clicking on the GitHub icon will allow you to add this, this will then provide a quick link to the discussion for you

Once you have created the support log create a new Discussion in GitHub and attach the zip file you downloaded. You can now added the GitHub reference in the WebUI if you wish

The manual method

This method should only be used if you are unable to use the automated method

The first step is to create a log file (/var/log/allsky.log) with the proper information to help us troubleshoot: 

sudo systemctl stop allsky               # Stop Allsky
sudo truncate -s 0 /var/log/allsky.log   # clears out log file
# Do NOT restart Allsky yet

Change Debug Level to 4 in the WebUI then click on the Save changes button to restart Allsky.

Wait until the problem occurs, then run: 

cp  /var/log/allsky.log  /tmp/allsky.log
cp  ~/allsky/config/settings.json  /tmp/settings.json.txt

Create a new Discussion in GitHub and attach /tmp/allsky.log and /tmp/settings.json.txt to it (do not copy/paste their text).
