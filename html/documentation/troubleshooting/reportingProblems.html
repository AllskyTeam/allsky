If you have a QUESTION or want to suggest a NEW FEATURE, add a new GitHub Discussion item; do not submit an Issue.

If you found a BUG or something isn't working right, first look at the existing Discussions and Issues. If the problem hasn't already been reported, follow the instructions below.

How to report a problem

The automated method

This is the preferred method and should be used if you have access to the WebUI.

Go to the Getting Support page in the WebUI to generage a troubleshooting log which you will attach to a GitHub discussion.

Follow the instructions on that page.

The manual method

This method should only be used if you are unable to use the automated method, for example, if Allsky didn't install and the WebUI isn't working.

If you can easily reproduce the problem, even if it means waiting until tomorrow for it to occur again, do the following:

  1. Run the following command to prepare Allsky for troubleshooting. It truncates the Allsky log files and makes sure the Debug Level is set to produce the necessary information to help the Allsky Team troubleshoot your problem: 
    allsky-config   prepare_logs
    If the Debug Level changed you are told what its prior value was so you can restore it when troubleshooting is over.
  2. Wait until the problem occurs, then follow the steps below.

Now:

  1. Run the following to create files to attach to a GitHub post: 
    cp  /var/log/allsky.log  /tmp/allsky.log
cp  ~/allsky/config/settings.json  /tmp/settings.json.txt
  2. Create a new Discussion in GitHub and attach /tmp/allsky.log and /tmp/settings.json.txt to it (do not copy/paste their text).
  3. Attach screenshots of any error messages or copy/paste their text and surround them with three tildas in the Discussion, for example: 
    ~~~
ERROR: error message line 1
  error message line 2, ...
~~~
    This makes the message format properly.
  4. If there are any other files you think would be helpful, attach those as well.
    GitHub only accepts certain file types. See this URL for a list of supported types. If your file has an unsupported extension, append a .txt to its name, e.g., settings.json.txt.
