This page lists known issues and limitations with the current Allsky release. Workarounds and plans to address the items, if known, are also given.

Some of items in the lists below are also in the list of possible changes to future Allsky versions.

Image Capture

  • Known Issue: The nighttime auto-exposure, auto-gain algorithm for ZWO cameras can produce flickering, especially if the sky is partially cloudy.
    Workaround: No good one, but you can try decreasing the Aggression setting in the WebUI which can smooth out the brightness changes, but is slower to react to true brightness changes.
    Future Plans: Port the RPi auto-exposure and auto-gain algorithm, which works well, to ZWO cameras.
  • Known Issue: Some ZWO cameras can produce ASI_ERROR_TIMEOUT messages and fail to take pictures.
    Workaround: See this page.
    Future Plans: We hope that when we port he RPi auto-exposure algorithm to ZWO it will fix this.
  • Limitation: RPi cameras don't support sensor temperature.
    Workaround: None.
    Future Plans: Will fix in the next release for users running the Bullseye operating system.
    The software Allsky uses to manage RPi cameras on the older Buster operating system is no longer being improved. All software updates for RPi cameras (like reading the sensor temperature) is happening only on Bullseye so we suggest upgrading to that operating system.
  • Limitation: RPi cameras don't easily support dark frames.
    Workaround: If you are technically savy and want to use dark frames submit a new Discussion asking the Allsky developers how to do it.
    Future Plans: Will fix in the next release for users running the Bullseye operating system.
  • Known Issue: Multiple timelapse creations can occur simultaneously - usually a full timelapse and "mini" timelapse, or two "mini" timelapse.
    Workaround: Decrease the frequency of "mini" timelapse creation.
    Future Plans: Will only allow one timelapse creation at a time.

Overlays

  • Known Issue: Fonts sometimes change to Glyphicons This just a display problem in the Text Editor; the actual font used is whatever you initially specified.
    Workaround: None
    Future Plans: Will fix in next release.

Modules

  • Limitation: Modules can only run once per flow, e.g., once per image, once per day/night transition, etc.
    Workaround: None
    Future Plans: Will change in next release.
  • Known Issue: The Overlay Module may be disabled the first time it runs. This is due to the module having to download the JPL ephemeris data which can take some time on a slow Internet connection.
    Workaround: Re-enable it in the Module Manager.
    Future Plans: Will fix in the next release.
  • Known Issue: The "additional modules" have been installed but they don't all appear.
    Workaround: Ensure the "Show Experimental" setting is enabled in the Module Manager's settings.
    Future Plans: Will fix in the next release.

Other

  • Limitation: Can't specify how images are sorted in the WebUI's Images page and in the Allsky Website.
    Workaround: None
    Future Plans: Will add options to specify ascending (a-z) or descending (z-a) order.
