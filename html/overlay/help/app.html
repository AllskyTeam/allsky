The Overlay Editor Basics
The editor allows you to control the data that is displayed on each frame captured by the AllSky software.
A drag and drop user iterfance allows for fields to be managed.
Data is displayed by adding 'fields' which define the data and format to be displayed, currently two field formats are available text and images.
Text fields allow textual data to be displayed and are dynamic, that is they can change between each captured frame. Images are static and do not change between frames.
Text fields are either 'System' or 'User' fields. System fields contain data defined by the AllSky software, such as the date or exposure duration. User fields conatin data provied by source sother than AllSky, an example might be weather data. User fields replace the 'extra text' feature in AllSky.
Text fields also support the use of 3rd party fonts. The editor has an inbuilt font manager where you can upload your own fonts.
The Field Manager
This provides a list of available fields and defines their format. Once a field has been created it can be added or its label used in composite fields, a composite field is one that consists of more than on field
The field manager is accessed by clicking on the 'field manager' icon in the toolbar. Once selected a list of existing fields is displayed. NOTE: The default page size for the list can be controlled in the overlay editor options.
When adding or editing a field the following fields are available
Name This is the label used for the field. The format needs to be specific ${VARIABLENAME}
Description The description for the field.
Format Fot non text fields this will define the format used for this field, see the formats section for more details
Sample If the field is a non system field then this is the sample data used when using switch the 'sample data' option.
Category The category the field belongs to, this is purley for display purposes
Type The type of field.
Source Either 'System' or 'User'. System fields cannot be deleted but can be edited.

Adding a simple field

Select the field manager and click the + button. The new field will be added to the overlay. Change the 'field' to show the data you require. Be careful NOT to modify the placeholders

The Font Manager

The overlay editor support additional fonts which are managed via the font manager. The Font Manager is accessed from the main toolbar.

The font manager will display all availabel fonts, both system and user uploaded fonts. The system fonts cannot be removed but it is possible to delete user fonts if they are no longer required.

The font manager ONLY supports fonts from daFont.com.

To upload a new font first ensure that the current config has been saved then select the 'Add Font' button from the Font Manager. Find the font you wish to upload on daFont.com and copy the url from the brower. NOTE: The url will always end in .font

Enter the font url in the dialog and the overlay editor will do the rest. If there is an issue installing the font an error will be displayed. If the font uploads ok then it will be available for use in fields.

To delete a font open the Font Manager and click the delete button. If any field is using the font it will revert to use whichever font is set as the default in the overlay editor options.
