If you have a QUESTION or want to suggest a NEW FEATURE, add a new Discussion item via the link at the top of any GitHub page; do not submit an Issue.

If you found a BUG or something isn't working right, first look in the Discussions area and at existing Issues. If no one has already reported the problem, create a new Issue.

We cannot debug a problem without the Allsky log file (/var/log/allsky.log) so if you want help you must attach it. The following commands create the proper log file: 

sudo systemctl stop allsky               # Stop the allsky service
sudo truncate -s 0 /var/log/allsky.log   # clears out log file
# Do NOT restart the service yet

Change Debug Level to 4 in the WebUI then click on the button. This will restart the service.

Wait until the problem occurs, then run sudo systemctl stop allsky to stop Allsky so the log file doesn't keep growing, making it hard to read. Then attach /var/log/allsky.log (do not copy/paste it) to your Issue in GitHub. If the problem may be related to your configuration, also attach ~/allsky/config/config.sh.

GitHub only accepts text files ending in .txt and .log. If your file has a different extension, append a .txt to it, i.e., config.sh.txt.

Do NOT zip your files; it makes it hard to read on a phone. If you feel you need to zip a file you probably didn't follow the instructions above.

In your description, please include the output of cd ~/allsky; git log | head.

When you create a new Issue, you will see a list of instructions and required information.
